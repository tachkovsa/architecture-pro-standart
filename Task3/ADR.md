Также опишите, какой логикой вы руководствовались в ходе принятия решений и выбора технологий. Не забывайте, что необходимо учесть все функциональные и нефункциональные требования.
1. Так как предполагается разработка MVP сервиса, было предложено выделить сервис обработки заявок, который бы взял на себя всю логику обработки входящих заявок на депозит, при этом не выдавая дополнительной нагрузки на монолитную АБС
2. Предлагается сразу внедрить Reverse Proxy между Desktop-клиентом и базой данных ABS для реализации более безопасной работы с базой данных ABS
3. Так как в перспективе возможно внедрение дополнительных сервисов, считаю целесообразным сразу внедрить API-шлюз на базе Kong для упращения внедрения новых сервисов в будущем - данный шлюз будет внедрять параллельно с разработкой нового сервиса
4. В связи с текущими ограничениями АБС, внедрение брокеров сообщений было исключено на текущем этапе - сервисы взаимодействуют синхронно посредством REST API
5. Решение о внедрении MongoDb принято для упрощения изменения схемы данных в будущем - заявка может сильно меняться в зависимости от множества факторов
